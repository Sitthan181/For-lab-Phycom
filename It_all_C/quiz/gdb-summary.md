# สรุปเนื้อหา GDB สำหรับเตรียมสอบ Quiz

## 1. การเตรียมไฟล์ก่อน Debug

โจทย์ Quiz แบบนี้ใช้โครงสร้างไฟล์เดียวกันเสมอ:

```c
#include "stdio.h"

int main() {
    int a = 17, b = 25, c = 50, d = 10;

    // Assignment Statements of Each Question

    return 0;
}
```

แต่ละข้อในตาราง = แทนที่ comment ด้วย statement ของข้อนั้น

---

## 2. Compile ด้วย flag `-g` (ห้ามลืม!)

```bash
gcc -g main.c -o main
```

- `-g` = ใส่ debugging information ลงไฟล์ที่ compile แล้ว
- `-o <file>` = กำหนดชื่อไฟล์ output
- **ถ้าลืม `-g` จะ debug ไม่ได้เลย** (ไม่มีข้อมูลเชื่อมโยงกับ source code)

---

## 3. เปิด GDB

```bash
gdb main
```

### เข้าโหมด TUI (แบ่งจอ เห็นโค้ดชัดเจน)

**วิธีที่ 1** — เปิดตรงๆ ด้วย flag (บางเครื่อง/บาง terminal เช่น mintty ของ Cygwin อาจมีปัญหาปิดตัวเองทันที):
```bash
gdb --tui main
```

**วิธีที่ 2** — เปิด gdb ปกติก่อน แล้วค่อยสลับโหมด (แนะนำ เพราะเสถียรกว่าบน Cygwin/mintty):
```bash
gdb main
```
จากนั้นกด:
```
Ctrl + x   แล้วตามด้วย   a
```
กด shortcut นี้ซ้ำได้เพื่อสลับเข้า/ออกโหมด TUI ตลอดเวลา

**วิธีที่ 3** — ใช้คำสั่ง `layout` ข้างใน gdb:
```gdb
(gdb) layout src
```

## 4. คำสั่งพื้นฐานที่ใช้บ่อยที่สุด

| คำสั่ง | ตัวย่อ | รูปแบบการใช้งาน | คำอธิบาย |
|---|---|---|---|
| **list** | `l` | `list` หรือ `l <line>` | แสดงโค้ด 10 บรรทัดรอบจุดปัจจุบัน หรือรอบบรรทัดที่ระบุ |
| **break** | `b` | `break <line>` หรือ `b <line>` | ตั้งจุดหยุด (breakpoint) ที่บรรทัดนั้น |
| **run** | `r` | `run` หรือ `r` | เริ่มรันโปรแกรมจนถึง breakpoint แรก หรือจบโปรแกรม |
| **print** | `p` | `print <var>` หรือ `p <var>` | แสดงค่าตัวแปร ณ ขณะนั้น |
| **next** | `n` | `next` หรือ `n` | รันบรรทัดถัดไป **ไม่เข้าไปในฟังก์ชัน** |
| **step** | `s` | `step` หรือ `s` | รันบรรทัดถัดไป **เข้าไปในฟังก์ชัน** ที่ถูกเรียก |
| **continue** | `c` | `continue` หรือ `c` | รันต่อเนื่องจนกว่าจะเจอ breakpoint ถัดไป หรือจบโปรแกรม |
| **finish** | `fin` | `finish` หรือ `fin` | รันจนจบฟังก์ชันปัจจุบัน แล้วหยุดที่บรรทัดถัดจากจุดเรียกฟังก์ชัน |
| **watch** | - | `watch <expr>` | หยุดโปรแกรมทันทีที่ค่าของตัวแปร/นิพจน์นั้นเปลี่ยนแปลง |
| **quit** | `q` | `quit` หรือ `q` | ออกจาก GDB |

## 5. คำสั่งจัดการ Breakpoint (สำคัญมากสำหรับ Quiz)

### `info breakpoints` (ตัวย่อ `i b`)
แสดงรายการ breakpoint/watchpoint ทั้งหมดที่ตั้งไว้ พร้อมรายละเอียด:

```gdb
(gdb) info breakpoints
```

ข้อมูลที่แสดง:
- **Num** — หมายเลขลำดับของ breakpoint
- **Type** — ประเภท (breakpoint / watchpoint)
- **Disp** (Disposition) — สถานะหลังถูกกระตุ้น (`keep` = เก็บไว้, `del` = ลบทิ้ง)
- **Enb** (Enabled) — เปิดใช้งานอยู่ไหม (y/n)
- **Address** — ตำแหน่งใน memory
- **What** — ไฟล์และเลขบรรทัดของ breakpoint นั้น

### `delete` (ตัวย่อ `d`)
ลบ breakpoint หรือ watchpoint ตามหมายเลข:

```gdb
(gdb) delete <num>
```

- ถ้าไม่ใส่หมายเลข (`delete` เฉยๆ) จะลบ **ทั้งหมด**
- ใช้ได้ทั้งกับ breakpoint และ watchpoint

### `break` แบบมีเงื่อนไข
```gdb
(gdb) break main if i == 10
```

### `info locals`
แสดงค่าตัวแปร local ทั้งหมดในสโคปปัจจุบันทีเดียว (ไม่ต้อง `print` ทีละตัว):
```gdb
(gdb) info locals
```

## 6. Workflow มาตรฐานสำหรับแต่ละ "ข้อ" Quiz (1 ข้อ = 1 ตาราง = อาจมีหลายแถว/statement)

⚠️ **สำคัญ**: ทุกแถวในตารางของข้อเดียวกันต้องเขียนรวมอยู่ใน**ไฟล์เดียวกัน รันต่อเนื่องกันตามลำดับ** (ดูรายละเอียดในหัวข้อ 7) จึงต้องแก้ไฟล์และ compile **แค่ครั้งเดียวต่อข้อ** ไม่ใช่ทุกแถว

1. แก้ไฟล์ด้วย `vim` ใส่ **ทุก statement ของทุกแถวในตารางข้อนั้น** แทน comment (เรียงตามลำดับแถวในตาราง)
2. บันทึกและ compile ใหม่ (`gcc -g main.c -o main`)
3. เปิด `gdb main` แล้วเข้า TUI (`Ctrl+x` → `a`)
4. ตั้ง breakpoint **ที่บรรทัดของทุกแถว/statement พร้อมกันตั้งแต่ต้น**:
   ```gdb
   (gdb) break <line1>
   (gdb) break <line2>
   (gdb) break <line3>
   ...
   (gdb) run
   ```
5. `print` หรือ `info locals` → ค่า **"ก่อน"** ของแถวที่ 1
6. `next` → `print`/`info locals` อีกครั้ง → ค่า **"หลัง"** ของแถวที่ 1
7. `continue` → โปรแกรมวิ่งไปหยุดที่ breakpoint แถวที่ 2 (ค่าตอนนี้ตรงกับค่า "หลัง" ของแถว 1 พอดี ใช้ตรวจทานได้)
8. `next` → `print`/`info locals` → ค่า **"หลัง"** ของแถวที่ 2
9. `continue` ไปแถวถัดไป ทำซ้ำจนครบทุกแถวในตาราง
10. `quit` ออกจาก gdb ก่อนไปทำข้อถัดไป (ข้อใหม่ = ตารางใหม่ = ต้องแก้ไฟล์และ compile ใหม่)

### ถ้าอยากรันซ้ำโดยไม่ออกจาก gdb (เช่น print ค่าก่อนเลื่อนหายจากจอ)

```gdb
(gdb) run
```
ถ้าโปรแกรมเคยรันไปแล้ว gdb จะถามยืนยัน:
```
The program being debugged has been started already.
Start it from the beginning? (y or n) y
```
พิมพ์ `y` แล้ว Enter จะกลับไปเริ่มต้นใหม่ หยุดที่ breakpoint เดิม (ไม่ต้องตั้งใหม่)

## 7. ⚠️ สำคัญมาก: Statement ในตารางเดียวกัน "รันต่อเนื่องกัน" ไม่ใช่แยกอิสระ

**กติกาจริงของโจทย์**: ทุก statement ที่อยู่ในตารางของข้อเดียวกัน ถือว่าอยู่ใน **โค้ดเดียวกัน และรันต่อกันตามลำดับ** (ไม่ได้ reset ค่าตัวแปรใหม่ทุกแถว) เช่น

```c
int a = 17, b = 25, c = 50, d = 10;

d = b % a + c / b;        // แถวที่ 1
d = b % a++ + c / b;      // แถวที่ 2 (ใช้ค่า a, b, c, d ต่อจากแถวที่ 1)
d = b % ++a + c / b;      // แถวที่ 3 (ใช้ค่าต่อจากแถวที่ 2)
d = b % (a + c) / b;      // แถวที่ 4 (ใช้ค่าต่อจากแถวที่ 3)

return 0;
```

**ดังนั้นค่า "ก่อน" ของแถวที่ 2 ต้องเท่ากับค่า "หลัง" ของแถวที่ 1 เสมอ** (ไล่ต่อกันไปเรื่อยๆ ทุกแถว) — **ห้ามตั้งชื่อตัวแปรใหม่แยกแต่ละแถว** (เช่น `a2`, `b2`) และ **ห้ามเปิด scope ใหม่ (`{ }`) เพื่อ "รีเซ็ต" ค่าตัวแปร** เพราะจะทำให้ค่าที่ได้ผิดจากที่โจทย์ต้องการ (โจทย์ต้องการเห็นผลสะสมของทุก statement ที่รันต่อกันมา ไม่ใช่แยกอิสระทีละ statement)

### วิธีเขียนโค้ดที่ถูกต้องสำหรับทั้งข้อ (รวมทุกแถวในไฟล์เดียว)

```c
#include "stdio.h"

int main() {
    int a = 17, b = 25, c = 50, d = 10;

    d = b % a + c / b;        // แถว 1
    d = b % a++ + c / b;      // แถว 2
    d = b % ++a + c / b;      // แถว 3
    d = b % (a + c) / b;      // แถว 4

    return 0;
}
```

### ตั้ง breakpoint ที่ทุกบรรทัดของ statement พร้อมกันตั้งแต่แรก

```gdb
(gdb) break 6
(gdb) break 7
(gdb) break 8
(gdb) break 9
(gdb) run
```

### Workflow ไล่ทีละแถว (ใช้ `continue` เชื่อมแต่ละแถว ไม่ใช่ `run` ใหม่)

1. หยุดที่ breakpoint แถวที่ 1 → `print` ทุกตัวแปร (ค่า "ก่อน" ของแถว 1)
2. `next` → `print` ทุกตัวแปร (ค่า "หลัง" ของแถว 1 = ค่า "ก่อน" ของแถว 2 ด้วย)
3. `continue` → โปรแกรมจะวิ่งไปหยุดที่ breakpoint แถวที่ 2 พอดี (ค่าที่เห็นตอนนี้ควรตรงกับค่า "หลัง" ของแถว 1 ที่เพิ่ง print ไป ใช้ตรวจทานความถูกต้องได้)
4. `next` → `print` (ค่า "หลัง" ของแถว 2)
5. `continue` ไปแถวที่ 3 → ทำซ้ำแบบเดิมไปเรื่อยๆ จนครบทุกแถวในตาราง

### เทคนิคลัด: ใช้ `info locals` แทนการ `print` ทีละตัว

แทนที่จะพิมพ์ `print a`, `print b`, `print c`, `print d` ทีละบรรทัด (4 คำสั่ง) ใช้คำสั่งเดียวจบ:

```gdb
(gdb) info locals
```

จะแสดงค่าตัวแปร local ทั้งหมดในสโคปปัจจุบันพร้อมกันในทีเดียว เช่น:
```
a = 17
b = 25
c = 50
d = 10
```

**ประหยัดเวลามากตอนสอบจริง** เพราะแต่ละแถวต้องเช็คค่า 4 ตัวแปร x 2 ครั้ง (ก่อน/หลัง) ถ้าใช้ `info locals` แทน `print` ทีละตัว จะลดจาก 8 คำสั่งเหลือแค่ 2 คำสั่งต่อแถว

**ข้อควรระวังของ `info locals`**: จะแสดง**เฉพาะตัวแปรที่อยู่ใน scope ปัจจุบัน ณ จุดที่โปรแกรมหยุดอยู่เท่านั้น** ถ้ามีการเปิด `{ }` ซ้อนในโค้ด (ซึ่งตามกติกาข้อนี้ไม่ควรมีอยู่แล้ว) ตัวแปรนอกบล็อกอาจไม่โชว์ครบ แต่ถ้าเขียนโค้ดแบบราบเรียบไม่มี scope ซ้อน (ตามคำแนะนำด้านบน) `info locals` จะโชว์ `a, b, c, d` ครบทุกตัวเสมอ

## 8. ข้อควรระวังทั่วไป

- ถ้าเจอข้อความค้าง `--Type <RET> for more, q to quit, c to continue without paging--` ให้กด `q` เพื่อข้ามออกมาก่อนเสมอ
- `gdb --version` (ติดกัน) กับ `gdb -- version` (มีช่องว่าง) ความหมายต่างกันโดยสิ้นเชิง — แบบมีช่องว่างจะพยายามเปิดไฟล์ชื่อ "version" มา debug แทน
- ทุกครั้งที่แก้โค้ดแล้ว **ต้อง compile ใหม่เสมอ** ก่อนเปิด gdb ไม่งั้นจะยังอ่านโค้ดเก่าอยู่
- ตัวเลข `$1, $2, $3...` ที่ gdb ใส่หน้าค่าที่ print ออกมา คือหมายเลขอ้างอิงค่าที่เก็บไว้ใน history ของ gdb เอง (ใช้เรียกกลับมาดูทีหลังได้ แต่ไม่จำเป็นต้องสนใจสำหรับ quiz นี้)
